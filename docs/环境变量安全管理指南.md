## 概述

环境变量包含敏感信息（数据库密码、API 密钥、JWT 密钥等），必须妥善管理以防止安全泄露。

### 🔴 当前存在的安全隐患

```bash
# ❌ 当前 .env 文件中的问题
JWT_SECRET="gametalent-secret-key-change-in-production-2026"  # 弱密钥
DATABASE_URL="postgresql://gametalent:gametalent_password@..." # 默认密码
SMTP_PASS="XdzLJ8YC1AArTz5p"                                    # 真实密码暴露
```

### ✅ 改进后的安全配置

```bash
# ✅ 安全的配置示例
JWT_SECRET="a1b2c3d4e5f6...64 chars of random hex..."  # 强随机生成
DATABASE_URL="postgresql://user:STRONG_RANDOM_PASSWORD@..."
SMTP_PASS="managed-by-secret-manager"                   # 密钥管理服务
```

## 安全风险

### 1. 密钥泄露风险

| 风险来源 | 影响 | 防护措施 |
|---------|------|---------|
| Git 仓库泄露 | 密钥永久暴露 | `.gitignore` + pre-commit hook |
| 日志泄露 | 日志中包含敏感信息 | 环境变量脱敏 |
| 前端暴露 | `NEXT_PUBLIC_*` 变量可见 | 最小化公开变量 |
| 服务器入侵 | 服务器上的 .env 文件 | 使用密钥管理服务 |

### 2. 常见攻击场景

```bash
# 场景 1: 开发者将 .env 提交到 Git
$ git add .env
$ git commit -m "Add env"
# ❌ 密钥永久暴露在 Git 历史中

# 场景 2: 日志中打印环境变量
console.log(process.env);  # ❌ 所有密钥泄露
logger.debug('Config:', config);  # ❌ 密钥记录到日志

# 场景 3: 前端代码使用敏感变量
const apiKey = process.env.API_KEY;  // ❌ 暴露在浏览器中
fetch('/api/data', {
  headers: { 'X-API-Key': apiKey }   # ❌ 任何人可见
});
```

## 最佳实践

### 1. 密钥强度要求

```typescript
// JWT 密钥长度要求
开发环境: 至少 32 字符
生产环境: 至少 64 字符（推荐使用 128 字符）

// 数据库密码要求
- 至少 16 字符
- 包含大小写字母、数字、特殊字符
- 不使用字典单词
- 定期轮换（每 90 天）
```

### 2. Git 安全配置

```bash
# .gitignore - 确保 .env 文件不被提交
.env
.env.local
.env.development.local
.env.test.local
.env.production.local

# 但保留模板文件
!.env.example
!.env.development.example
!.env.production.example
```

### 3. Pre-commit Hook

```bash
# .husky/pre-commit
#!/bin/sh
. "$(dirname "$0")/_/husky.sh"

# 检查是否意外提交了 .env 文件
if git diff --cached --name-only | grep -E "^\.env$"; then
  echo "❌ 检测到 .env 文件被提交！"
  echo "请将 .env 添加到 .gitignore 并取消此次提交。"
  exit 1
fi

# 检查代码中是否包含敏感信息
if git diff --cached | grep -E "(JWT_SECRET|DATABASE_PASSWORD|API_KEY)"; then
  echo "⚠️  警告: 检测到可能的敏感信息！"
  echo "请确认这不是真实的密钥。"
fi
```

### 4. 环境变量分层

```
项目根目录/
├── .env.example                    # 公开模板（无真实值）
├── .env.development.example        # 开发环境模板
├── .env.production.example         # 生产环境模板
├── .env.local                      # 本地开发（不提交）
└── .env.production.local           # 本地生产测试（不提交）
```

### 5. 日志脱敏

```typescript
// config/logger.ts
import { inspect } from 'util';

// 敏感字段列表
const SENSITIVE_FIELDS = [
  'password',
  'secret',
  'token',
  'apiKey',
  'apiSecret',
  'jwtSecret',
  'smtpPass',
  'databasePassword',
];

/**
 * 脱敏函数
 */
function redact(obj: any): any {
  if (typeof obj !== 'object' || obj === null) {
    return obj;
  }

  return Object.keys(obj).reduce((acc, key) => {
    const lowerKey = key.toLowerCase();

    // 检查是否是敏感字段
    const isSensitive = SENSITIVE_FIELDS.some(field =>
      lowerKey.includes(field.toLowerCase())
    );

    if (isSensitive) {
      acc[key] = '***REDACTED***';
    } else if (typeof obj[key] === 'object') {
      acc[key] = redact(obj[key]);
    } else {
      acc[key] = obj[key];
    }

    return acc;
  }, {});
}

// 使用示例
const config = {
  database: {
    url: 'postgresql://user:password@host/db',
  },
  jwt: {
    secret: 'my-secret-key',
  },
};

console.log(redact(config));
// 输出: { database: { url: 'postgresql://user:***@host/db' }, jwt: { secret: '***' } }
```

```bash

# 仅生成 API 密钥
npx ts-node scripts/generate-secrets.ts --api
```

### 手动生成（使用 OpenSSL）

```bash
# JWT 密钥 (64 字符 hex)
openssl rand -hex 32

# 数据库密码 (24 字符)
openssl rand -base64 24 | tr -d "=+/" | cut -c1-24

# API 密钥
openssl rand -base64 32 | tr -d "=+/" | cut -c1-32
```

```bash
# 1. 复制开发环境模板
cp .env.development.example .env.local

npx ts-node scripts/generate-secrets.ts
```

### 生产环境设置

```bash
# 1. 生成强随机密钥
npx ts-node scripts/generate-secrets.ts --all

# 2. 将密钥存储到密钥管理服务
#    (AWS Secrets Manager / Google Secret Manager / HashiCorp Vault)

export DATABASE_URL="your-production-db-url"

# 4. 或使用 .env.production.local（不提交到 Git）
cp .env.production.example .env.production.local
# 填入从密钥管理服务获取的值

# 5. 启动应用
npm run start:prod
```

## 密钥管理服务

### AWS Secrets Manager

```typescript
// 安装 AWS SDK
npm install @aws-sdk/client-secrets-manager

// apps/api/src/config/secrets-manager.ts
import {
  SecretsManagerClient,
  GetSecretValueCommand,
} from '@aws-sdk/client-secrets-manager';

const client = new SecretsManagerClient({ region: 'us-east-1' });

export async function getSecret(secretName: string): Promise<any> {
  const response = await client.send(
    new GetSecretValueCommand({ SecretId: secretName })
  );

  if (response.SecretString) {
    return JSON.parse(response.SecretString);
  }

  throw new Error('Secret not found');
}

// 使用
const secrets = await getSecret('gametalent/prod');
process.env.JWT_SECRET = secrets.jwt_secret;
process.env.DATABASE_URL = secrets.database_url;
```

### Google Secret Manager

```typescript
// 安装 Google Cloud SDK
npm install @google-cloud/secret-manager

import { SecretManagerServiceClient } from '@google-cloud/secret-manager';

const client = new SecretManagerServiceClient();

export async function getSecret(secretName: string): Promise<string> {
  const [version] = await client.accessSecretVersion({
    name: `projects/PROJECT_ID/secrets/${secretName}/versions/latest`,
  });

  return version.payload?.data?.toString() || '';
}

// 使用
const jwtSecret = await getSecret('jwt-secret');
process.env.JWT_SECRET = jwtSecret;
```

### HashiCorp Vault

```typescript
// 安装 Vault 客户端
npm install node-vault

import vault from 'node-vault';

const client = vault({
  endpoint: 'https://vault.your-domain.com',
  token: process.env.VAULT_TOKEN,
});

export async function getSecret(path: string): Promise<any> {
  return client.read(path);
}

// 使用
const secrets = await getSecret('secret/data/gametalent/prod');
process.env.JWT_SECRET = secrets.data.data.jwt_secret;
```

### 环境变量验证流程

```typescript
// apps/api/src/main.ts
import { validate } from './config/env.validation';

async function bootstrap() {
  try {
    // 1. 验证环境变量
    console.log('🔍 验证环境变量...');
    const config = validate(process.env);

    console.error('❌ 应用启动失败:', error.message);
    process.exit(1);
  }
}

bootstrap();
```

## 故障排除

### 问题 1: 环境变量未生效

**症状**: 更新 .env 文件后，应用仍使用旧值

**原因**: 环境变量在应用启动时加载，修改后需要重启

**解决方案**:
```bash
# 重启应用
npm run dev

# 或使用 nodemon 自动重启
npm run dev:watch
```

### 问题 2: 生产环境密钥泄露

**症状**: 发现 GitHub 仓库中包含真实密钥

**立即处理**:
```bash
# 1. 立即轮换所有密钥
npx ts-node scripts/generate-secrets.ts --all

# 2. 从 Git 历史中清除
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env" \
  --prune-empty --tag-name-filter cat -- --all

# 3. 强制推送
git push origin --force --all

# 4. 通知所有相关人员
# 5. 监控异常访问
```

### 问题 3: Docker 环境变量未传递

**症状**: Docker 容器中的应用读取不到环境变量

**解决方案**:
```yaml
# docker-compose.yml
services:
  api:
    env_file:
      - .env          # ✅ 加载所有变量
    environment:
      - NODE_ENV=production  # ✅ 覆盖特定变量
```
