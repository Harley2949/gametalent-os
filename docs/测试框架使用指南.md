## 概述

本项目采用**三层测试金字塔**架构：

```
        /\
       /  \      E2E 测试 (Playwright)
      /____\     - 关键用户流程
     /      \    - 集成测试
    /        \
   /__________\  单元测试 (Jest)
  /            \ - 后端服务
 /              \ - 前端组件/ Hooks
/________________\
```

### 测试覆盖范围

| 测试类型 | 工具 | 覆盖范围 | 执行时间 |
|---------|------|---------|---------|
| 单元测试 | Jest | 组件、Hooks、Services | 秒级 |
| 集成测试 | Jest | API 模块、数据流 | 秒级 |
| E2E 测试 | Playwright | 用户流程 | 分钟级 |

## 快速开始

### 1. 安装测试依赖

```bash
# 安装所有依赖（包括测试依赖）
pnpm install

# 单独安装测试依赖
pnpm add -D @playwright/test
pnpm add -D @testing-library/react @testing-library/jest-dom
pnpm add -D jest-environment-jsdom
```

### 2. 初始化 Playwright 浏览器

```bash
npx playwright install
```

### 3. 运行所有测试

```bash
# 后端单元测试
cd apps/api
npm run test

# 前端单元测试
cd apps/web
npm run test

# E2E 测试
cd ../..
npx playwright test
```

## 后端测试

### 单元测试

测试单个服务和函数的逻辑。

**示例：认证服务测试**

```typescript
// apps/api/test/auth.service.spec.ts
describe('AuthService', () => {
  it('应该成功登录有效用户', async () => {
    const result = await service.login({
      email: 'test@example.com',
      password: 'password123',
    });

    expect(result.access_token).toBeDefined();
    expect(result.user.email).toBe('test@example.com');
  });
});
```

### 运行后端测试

```bash
cd apps/api

# 运行所有测试
npm run test

# 监听模式（开发时使用）
npm run test:watch

# 生成覆盖率报告
npm run test:cov

# 运行特定测试文件
npm run test auth.service.spec.ts

# 运行匹配模式的测试
npm run test -- --testNamePattern="登录"
```

### 后端测试工具函数

使用 `test-helpers.ts` 中的工具函数简化测试编写：

```typescript
import { createTestCandidate, cleanupTestData } from '../utils/test-helpers';

describe('CandidatesService', () => {
  it('应该创建候选人', async () => {
    const candidate = await createTestCandidate(prisma, {
      email: 'test@example.com',
      name: '测试用户',
    });

    expect(candidate.id).toBeDefined();

    // 测试后清理
    await cleanupTestData(prisma, {
      emails: [candidate.email],
    });
  });
});
```

## 前端测试

### 组件测试

测试 React 组件的渲染和交互。

**示例：BackButton 组件测试**

```typescript
// apps/web/src/components/__tests__/BackButton.test.tsx
import { render, screen, fireEvent } from '@testing-library/react'
import BackButton from '../BackButton'

describe('BackButton 组件', () => {
  it('应该渲染返回按钮', () => {
    render(<BackButton>返回</BackButton>)
    expect(screen.getByRole('button')).toBeInTheDocument()
  })

  it('点击时应该调用 router.back()', () => {
    const mockBack = jest.fn()
    jest.mock('next/navigation', () => ({
      useRouter: () => ({ back: mockBack }),
    }))

    render(<BackButton>返回</BackButton>)
    fireEvent.click(screen.getByRole('button'))

    expect(mockBack).toHaveBeenCalled()
  })
})
```

### Hook 测试

测试自定义 Hook 的逻辑和状态管理。

```typescript
// apps/web/src/hooks/__tests__/useAuth.test.ts
import { renderHook, act, waitFor } from '@testing-library/react'
import { useAuth } from '../useAuth'

describe('useAuth Hook', () => {
  it('应该成功登录并设置用户', async () => {
    const { result } = renderHook(() => useAuth())

    await act(async () => {
      await result.current.login('test@example.com', 'password123')
    })

    expect(result.current.user).toBeTruthy()
    expect(result.current.error).toBeNull()
  })
})
```

## E2E 测试

端到端测试验证完整的用户流程。

### 运行 E2E 测试

```bash
# 运行所有 E2E 测试
npx playwright test

# 交互式 UI 模式（推荐）
npx playwright test --ui

# 调试模式
npx playwright test --debug

# 运行特定测试
npx playwright test auth.spec.ts

# 在特定浏览器运行
npx playwright test --project=chromium

# 查看测试报告
npx playwright show-report
```

### E2E 测试示例

```typescript
// e2e/auth.spec.ts
test('应该成功登录', async ({ page }) => {
  await page.goto('/login')

  await page.fill('input[type="email"]', 'admin@gametalent.os')
  await page.fill('input[type="password"]', 'admin123')
  await page.click('button[type="submit"]')

  await page.waitForURL('/dashboard')
  await expect(page.locator('h1')).toContainText('仪表板')
})
```

### Playwright 常用命令

```typescript
// 导航
await page.goto('/candidates')
await page.goBack()
await page.goForward()

// 查找元素
await page.locator('button').click()
await page.click('text=提交')
await page.fill('input[name="email"]', 'test@example.com')

// 等待
await page.waitForURL('/dashboard')
await page.waitForSelector('table')
await page.waitForTimeout(1000)

// 断言
await expect(page).toHaveURL('/dashboard')
await expect(page.locator('h1')).toContainText('候选人')
await expect(page.locator('button')).toBeVisible()
```

### 3. 遵循 AAA 模式

```typescript
it('应该返回候选人列表', async () => {
  // Arrange（准备）
  const mockCandidates = [ /* ... */ ]
  jest.spyOn(service, 'findAll').mockResolvedValue(mockCandidates)

  // Act（执行）
  const result = await service.findAll()

  // Assert（断言）
  expect(result).toEqual(mockCandidates)

```

### 4. 避免测试实现细节


## 故障排除

### 问题 1：后端测试数据库连接失败

**错误信息**：`Can't reach database server`

**解决方案**：
```bash
# 确保 Docker PostgreSQL 正在运行
docker compose up -d postgres

# 验证数据库连接
pnpm db:studio
```

### 问题 2：前端测试 "Module not found"

**错误信息**：`Cannot find module '@gametalent/ui'`

**解决方案**：
```bash
# 重新构建依赖
pnpm install
pnpm --filter @gametalent/ui build
```

### 问题 3：E2E 测试浏览器未安装

**错误信息**：`Executable doesn't exist`

**解决方案**：
```bash
# 安装 Playwright 浏览器
npx playwright install

# 或安装特定浏览器
npx playwright install chromium
```

### 问题 4：测试超时

**错误信息**：`Test timeout of 5000ms exceeded`

**解决方案**：
```typescript
// 增加超时时间
test('慢速测试', async ({ page }) => {
  test.setTimeout(60000) // 60 秒
  // ...
})
```

### 问题 5：Mock 未生效

**解决方案**：
```typescript
// 确保在导入模块前进行 mock
jest.mock('../module')

// 或清除模块缓存
beforeEach(() => {
  jest.resetModules()
})
```

## 持续集成 (CI)

### GitHub Actions 示例

```yaml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 18

      - name: Install pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8

      - name: Install dependencies
        run: pnpm install

      - name: Install Playwright browsers
        run: npx playwright install --with-deps

      - name: Run tests
        run: |
          pnpm --filter @gametalent/api test
          pnpm --filter @gametalent/web test
          npx playwright test

      - name: Upload coverage reports
        uses: codecov/codecov-action@v3
```
