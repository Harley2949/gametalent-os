## 详细设计

### 1. 候选人管理模块

#### 数据库（已有，需验证）
```prisma
model Candidate {
  id              String   @id @default(cuid())
  email           String   @unique
  name            String
  phoneNumber     String?
  avatar          String?
  status          CandidateStatus @default(ACTIVE)
  source          CandidateSource
  sourceUrl       String?
  linkedinUrl     String?
  githubUrl       String?
  portfolioUrl    String?
  location        String?
  currentCompany  String?
  currentTitle    String?
  expectedSalary  String?
  noticePeriod    String?
  yearsOfExperience Int?
  notes           String?
  tags            String[]
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  resumes         Resume[]
  applications    Application[]
}
```

#### API 接口
```
GET    /candidates              列表（搜索、筛选、分页）
GET    /candidates/:id          详情
POST   /candidates              创建
PUT    /candidates/:id          更新
DELETE /candidates/:id          删除
POST   /candidates/:id/resumes  上传简历
GET    /candidates/:id/match    匹配职位（AI）
```

#### 前端页面
```
/candidates
/candidates/:id
/candidates/new
/candidates/:id/edit
```

### 3. 应聘流程模块

```

```

```
PUT    /applications/:id/status   更新状态
GET    /applications/:id/feedback  获取反馈
POST   /applications/:id/offer    发送 Offer
```

#### 前端页面
```
/applications
/applications/:id
```

### 5. AI 简历解析模块

#### 技术方案
- **MVP 阶段**：使用 OpenAI API 或阿里云通义千问
- **后续阶段**：本地部署 Ollama

#### 解析能力
- 基础信息提取（姓名、联系方式、工作经历）
- 技能标签识别（技术栈、工具）
- 游戏行业专属解析（引擎、项目类型、美术风格）
- 简历真实性检测（工作时长异常、技能与经历不匹配）

#### API 接口
```
POST   /ai/parse-resume          解析简历文件
POST   /ai/extract-skills        提取技能
POST   /ai/analyze-experience    分析工作经历
```

## 数据库迁移

### 需要执行的迁移命令

```bash
# 1. 合并所有 schema 文件
cd apps/api
npm run schema:merge

# 2. 生成 Prisma Client
npm run prisma:generate

# 3. 推送数据库变更
npm run prisma:push

# 或者使用迁移（推荐生产环境）
```
