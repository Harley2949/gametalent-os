# 简历解析优化 - 20260317

## 问题描述

用户上传简历后显示"解析成功"，但所有字段都是空的：
- 姓名 = "待完善"
- 邮箱 = "无"
- 技能 = 0

## 根本原因分析

1. **AI 响应解析失败**：直接对 AI 返回结果进行 `JSON.parse()`，但 AI 可能返回 JSON + 解释文字
2. **缺少调试日志**：无法看到 AI 实际返回了什么内容
3. **Prompt 不够明确**：没有提供具体的 JSON 结构模板
4. **字段映射错误**：`portfolio.github` 被当作 `data.github`
5. **空值处理不当**：空姓名直接创建候选人，而不是生成默认名称

## 测试验证

### 成功场景
- ✅ AI 正常返回 JSON → 解析成功，所有字段填充
- ✅ AI 返回 JSON + 解释 → 自动提取 JSON，解析成功
- ✅ AI 返回 ```json...``` 代码块 → 自动提取，解析成功

### 降级场景
- ✅ AI 超时 → 使用正则提取基础信息
- ✅ AI 返回无效 JSON → 使用正则提取
- ✅ 简历为扫描件（OCR 失败）→ 返回警告，允许手动填写

### 边界场景
- ✅ 简历无姓名 → 生成"候选人1234"或邮箱前缀
- ✅ 简历无邮箱 → 允许创建，标记为"无"
- ✅ 简历无手机 → 正常创建，手机号为空
