**操作步骤**:
1. 鼠标悬停在任何应聘卡片上
2. 观察卡片变化（应该有阴影和轻微放大）
3. 按住鼠标左键拖动卡片
4. 拖动到另一个状态列
5. 观察目标列的视觉反馈
6. 释放鼠标

**预期结果**:
- ✅ 鼠标悬停时：卡片显示阴影 + 轻微放大效果
- ✅ 开始拖拽时：卡片变半透明 + 缩小到 95%
- ✅ 拖拽到目标列时：目标列显示蓝色边框 + 放大 105%
- ✅ 释放后：卡片移动到新列
- ✅ 更新时：显示蓝色旋转加载图标
- ✅ 原列和新列的数量徽章更新

#### 4. 测试视觉反馈
**预期状态**:

| 状态 | 卡片样式 | 目标列样式 |
|------|---------|-----------|
| 正常 | 无特殊样式 | 无特殊样式 |
| 悬停 | 阴影 + scale-[1.02] | - |
| 拖拽中 | opacity-50 + scale-95 | - |
| 拖拽悬停 | opacity-50 + scale-95 | ring-2 ring-blue-500 + scale-105 |

#### 5. 测试错误处理
**测试场景**:
1. 拖拽到同一列（状态不变）
2. 快速连续拖拽
3. 拖拽后立即切换到列表视图

**预期结果**:
- ✅ 拖拽到同一列：不做任何操作，卡片回到原位
- ✅ API 失败时：显示错误提示，数据重新加载
- ✅ 更新中：卡片显示禁用状态，无法再次拖拽

#### 6. 测试状态流转
**验证以下状态流转是否正常**:
```
APPLIED → SCREENING → REVIEWED → SHORTLISTED → INTERVIEWING
APPLIED → REJECTED (直接拒绝)
```
