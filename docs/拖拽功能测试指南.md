## 🔍 可能的问题和解决方案

### 问题 1：提示"请先登录"
**原因**：localStorage 中没有 token

**解决方法**：
1. 访问 http://localhost:3000/login
2. 重新登录
3. 返回看板页面再试

### 问题 2：提示"登录已过期"
**原因**：Token 已失效（一般 7 天后过期）

**解决方法**：
1. 点击页面右上角的"退出登录"
2. 重新登录
3. 返回看板页面

### 问题 3：提示"无法从 X 转换到 Y"
**原因**：后端的状态转换规则限制了某些状态变更

**解决方法**：
- 尝试拖拽到其他状态列
- 这是正常的业务逻辑限制

### 问题 4：拖拽时没有高亮效果
**原因**：可能是浏览器兼容性问题

**解决方法**：
- 使用 Chrome、Edge 或 Firefox 最新版本
- 刷新页面重试

### 问题 5：数据没有保存
**原因**：API 调用失败

**检查方法**：
1. 打开浏览器开发者工具（F12）
2. 切换到 Console 标签
3. 查看是否有错误信息
4. 切换到 Network 标签
5. 查看是否有失败的 API 请求

### 方法 3：API 验证
```bash
# 获取所有应聘记录
TOKEN=$(curl -s -X POST http://localhost:3006/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@gametalent.os","password":"admin123"}' \
  | grep -o '"access_token":"[^"]*"' | cut -d'"' -f4)

curl -s http://localhost:3006/api/applications \
  -H "Authorization: Bearer $TOKEN" \
  | grep -o '"status":"[^"]*"' | sort | uniq -c
```

### 查看网络请求
1. 打开开发者工具（F12）
2. 切换到 Network 标签
3. 拖拽一个卡片
4. 查找 `PUT /api/applications/{id}/status` 请求
5. 点击查看详情：
   - Request Headers：应该有 Authorization 字段
   - Request Payload：应该有新的 status
   - Response：应该返回成功状态
