# 统一错误处理迁移指南

## 概述

本文档说明如何将现有的 API 调用迁移到新的统一错误处理方案。

## 新增文件

1. **`apps/web/src/lib/errors.ts`** - 错误类型定义
2. **`apps/web/src/lib/fetchClient.ts`** - API 请求拦截器
3. **`apps/web/src/hooks/useErrorHandler.ts`** - 错误处理 Hook
4. **`apps/web/src/lib/api-refactored.example.ts`** - 重构示例

## 迁移步骤

### 步骤 1：更新 API 函数

**重构前**（`apps/web/src/lib/api.ts`）:
```typescript
export async function deleteJob(id: string) {
  const response = await fetch(`${API_BASE_URL}/api/jobs/${id}`, {
    method: 'DELETE',
  });

  if (!response.ok) {
    throw new Error('删除职位失败');
  }

  return { success: true };
}
```

**重构后**:
```typescript
import { del } from './fetchClient';

export async function deleteJob(id: string) {
  return del(`/api/jobs/${id}`);
}
```

### 步骤 2：更新组件中的错误处理

**重构前**:
```typescript
const handleDelete = async (id: string) => {
  try {
    await deleteJob(id);
    toast.success('删除成功');
  } catch (error) {
    console.error(error);
    toast.error('删除失败');
  }
};
```

**重构后**:
```typescript
const { withErrorHandling } = useErrorHandler();

const handleDelete = async (id: string) => {
  await withErrorHandling(
    () => deleteJob(id),
    {
      showSuccessToast: true,
      successMessage: '删除成功',
    }
  );
};
```

## 错误类型

系统会自动识别以下错误类型：

| 错误类型 | HTTP 状态码 | 说明 |
|---------|------------|------|
| `NetworkError` | 0 | 网络连接失败 |
| `AuthError` | 401, 403 | 认证/授权失败 |
| `NotFoundError` | 404 | 资源不存在 |
| `BusinessError` | 400, 422 | 业务逻辑错误 |
| `TimeoutError` | 408 | 请求超时 |
| `ServerError` | 500, 502, 503 | 服务器错误 |

## 高级用法

### 1. 自定义错误消息

```typescript
throw new BusinessError('自定义错误消息', { field: 'email' });
```

### 2. 跳过认证

```typescript
get('/api/public/data', { skipAuth: true });
```

### 3. 自定义超时

```typescript
post('/api/slow-endpoint', data, { timeout: 60000 }); // 60秒
```

### 4. 文件上传

```typescript
import { upload } from './fetchClient';

const result = await upload('/api/upload', file);
```

### 5. 并发请求

```typescript
const [data1, data2] = await Promise.all([
  get('/api/endpoint1'),
  get('/api/endpoint2'),
]);
```

## 常见问题

### Q: 如何禁用某个请求的错误提示？

```typescript
post('/api/endpoint', data, { skipErrorToast: true });
```

### Q: 如何在服务端组件中使用？

服务端组件无法使用 `useErrorHandler` Hook，需要手动处理：

```typescript
import { fetchClient } from './fetchClient';
import { getErrorMessageSafely } from './hooks/useErrorHandler';

export default async function ServerComponent() {
  try {
    const data = await fetchClient('/api/data');
    return <div>{JSON.stringify(data)}</div>;
  } catch (error) {
    const message = getErrorMessageSafely(error);
    return <div>Error: {message}</div>;
  }
}
```

### Q: 如何处理 Mock 数据降级？

新方案不再支持自动降级到 Mock 数据。建议使用环境变量或特性开关：

```typescript
const USE_MOCK = process.env.NEXT_PUBLIC_USE_MOCK === 'true';

export async function fetchJobs() {
  if (USE_MOCK) {
    return mockJobs;
  }
  return get('/api/jobs');
}
```
