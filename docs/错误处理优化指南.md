## 后端错误处理

### 1. 统一错误类

**文件**：[apps/api/src/common/errors/app.error.ts](apps/api/src/common/errors/app.error.ts)

#### 基础错误类

```typescript
import { AppError, ErrorCode } from '@gametalent/api/common/errors';

// 抛出 404 错误
throw AppError.notFound('候选人', { id: '123' });

// 抛出 400 验证错误
throw AppError.validation('邮箱格式无效', {
  field: 'email',
  value: 'invalid-email',
});

// 抛出 401 未授权错误
throw AppError.unauthorized('请先登录');

// 抛出 403 权限错误
throw AppError.forbidden('您没有权限执行此操作');

// 抛出 409 冲突错误
throw AppError.conflict('该邮箱已被注册');

// 抛出业务规则错误
throw new BusinessRuleError('不能删除有关联数据的候选人');
```

### 3. Service 中使用错误类

```typescript
import { AppError, ErrorCode } from '@gametalent/api/common/errors';

@Injectable()
export class CandidatesService {
  async findOne(id: string) {
    const candidate = await this.prisma.candidate.findUnique({
      where: { id },
    });

    if (!candidate) {
      throw AppError.notFound('候选人', { id });
    }

    return candidate;
  }

  async create(data: CreateCandidateDto) {
    // 检查邮箱是否已存在
    const existing = await this.prisma.candidate.findUnique({
      where: { email: data.email },
    });

    if (existing) {
      throw AppError.conflict('该邮箱已被使用', {
        email: data.email,
      });
    }

    return this.prisma.candidate.create({ data });
  }

  async delete(id: string) {
    const candidate = await this.findOne(id);

    // 检查业务规则
    if (candidate.applications.length > 0) {
      throw new BusinessRuleError(
        '无法删除：该候选人有关联的应聘记录',
        {
          candidateId: id,
          applicationCount: candidate.applications.length,
        }
      );
    }

    await this.prisma.candidate.delete({ where: { id } });
  }
}
```

### 4. Controller 中处理验证错误

```typescript
import { ValidationError } from '@gametalent/api/common/errors';

@Controller('candidates')
export class CandidatesController {
  @Post()
  async create(@Body() dto: CreateCandidateDto) {
    // 手动验证
    const errors = await validate(dto);
    if (errors.length > 0) {
      throw AppError.validation('请求参数验证失败', {
        errors: formatValidationErrors(errors),
      });
    }

    return this.candidatesService.create(dto);
  }
}
```

## 前端错误处理

### 1. 统一错误类

**文件**：[apps/web/src/lib/errors.ts](apps/web/src/lib/errors.ts)

#### 基础使用

```typescript
import {
  ApiError,
  NetworkError,
  AuthError,
  BusinessError,
  getUserFriendlyMessage,
} from '@/lib/errors';

try {
  const response = await fetch('/api/candidates/123');
  if (!response.ok) {
    await handleApiErrorResponse(response);
  }
  const data = await response.json();
} catch (error) {
  if (error instanceof AuthError) {
    // 处理认证错误
    console.log('用户需要登录');
  } else if (error instanceof NetworkError) {
    // 处理网络错误
    console.log('网络连接失败');
  } else {
    // 获取用户友好的错误消息
    const message = getUserFriendlyMessage(error);
    console.error(message);
  }
}
```

### 2. 使用 safeFetch 包装 API 请求

```typescript
import { safeFetch, getUserFriendlyMessage } from '@/lib/errors';

async function fetchCandidate(id: string) {
  try {
    const response = await safeFetch(`/api/candidates/${id}`);
    return response.json();
  } catch (error) {
    const message = getUserFriendlyMessage(error);
    toast.error(message);
    throw error;
  }
}
```

### 4. 表单验证错误处理

```typescript
import { extractFieldErrors, ApiError } from '@/lib/errors';

async function handleSubmit(data: FormData) {
  try {
    await createCandidate(data);
    toast.success('创建成功');
  } catch (error) {
    if (error instanceof ApiError) {
      const fieldErrors = extractFieldErrors(error);

      if (fieldErrors) {
        // 设置表单字段的错误
        Object.entries(fieldErrors).forEach(([field, message]) => {
          form.setError(field, { type: 'manual', message });
        });
      } else {
        // 显示通用错误消息
        toast.error(error.message);
      }
    }
  }
}
```

## 最佳实践

### 1. 错误处理原则

✅ **DO**:
- 使用统一的错误类
- 提供用户友好的错误消息
- 记录详细的错误日志
- 区分客户端错误（4xx）和服务器错误（5xx）
- 使用适当的 HTTP 状态码

❌ **DON'T**:
- 不要暴露敏感信息（密码、密钥等）
- 不要在生产环境返回堆栈跟踪
- 不要忽略错误（空的 catch 块）
- 不要使用通用错误消息（"出错了"）
- 不要在错误消息中包含技术细节

### 2. HTTP 状态码使用指南

| 状态码 | 场景 | 示例 |
|-------|------|------|
| 400 | 请求参数错误 | 邮箱格式无效 |
| 401 | 未认证 | Token 过期、未登录 |
| 403 | 无权限 | 普通用户访问管理员功能 |
| 404 | 资源不存在 | 候选人 ID 不存在 |
| 409 | 资源冲突 | 邮箱已被注册 |
| 422 | 验证失败 | 必填字段缺失 |
| 429 | 请求过于频繁 | 超过速率限制 |
| 500 | 服务器错误 | 数据库连接失败 |
| 503 | 服务不可用 | 维护模式 |

### 3. 错误消息编写指南

#### ❌ 不好的错误消息

```typescript
// 太模糊
"错误"
"操作失败"
"Invalid input"

// 太技术化
"SQL query failed: UNIQUE constraint failed"
"NullPointerException at line 42"

// 暴露敏感信息
"Password '123456' is too weak"
"Database connection failed: root@localhost"
```

#### ✅ 好的错误消息

```typescript
// 清晰具体
"邮箱格式不正确，请输入有效的邮箱地址"
"候选人不存在或已被删除"
"该邮箱已被注册，请使用其他邮箱或登录"

// 提供解决建议
"登录已过期，请重新登录"
"文件过大，请选择小于 5MB 的文件"
"网络连接失败，请检查网络设置后重试"

// 用户友好
"操作成功"
"保存失败，请稍后重试"
```

### 4. 异步错误处理

```typescript
// ✅ 使用 try-catch
async function fetchData() {
  try {
    const data = await apiCall();
    return data;
  } catch (error) {
    handleError(error);
    throw error;
  }
}

// ✅ 使用 Promise.catch
async function fetchData() {
  return apiCall().catch(handleError);
}

// ❌ 不要忽略错误
async function fetchData() {
  try {
    return await apiCall();
  } catch (error) {
    // 空的 catch 块
  }
}
```

## 故障排除

### 问题 1：全局过滤器未生效

**症状**：错误返回的是默认的 HTML 格式，不是 JSON

**解决方案**：

1. 确保在 `main.ts` 中注册了过滤器：
```typescript
app.useGlobalFilters(new AllExceptionsFilter());
```

2. 确保没有其他过滤器覆盖它：
```typescript
// 检查是否有多个过滤器注册
app.useGlobalFilters(new HttpExceptionFilter());
app.useGlobalFilters(new AllExceptionsFilter());  // 应该是最后一个
```

### 问题 2：前端无法解析错误响应

**症状**：`SyntaxError: Unexpected token < in JSON`

**原因**：服务器返回的是 HTML 错误页面，不是 JSON

**解决方案**：

确保后端使用全局过滤器，所有错误都返回 JSON 格式。

### 问题 3：ErrorBoundary 未捕获错误

**症状**：组件报错后没有显示错误 UI

**原因**：
- Error events（如图片加载失败）
- 异步代码（setTimeout、Promise）
- 服务端渲染
- 事件处理器中的错误

**解决方案**：

```typescript
// 对于事件处理器中的错误
handleClick() {
  try {
    riskyOperation();
  } catch (error) {
    this.setState({ error });
  }
}

// 对于异步错误
useEffect(() => {
  const fetchData = async () => {
    try {
      const data = await fetch('/api/data');
      // ...
    } catch (error) {
      // 处理错误
    }
  };

  fetchData();
}, []);
```
