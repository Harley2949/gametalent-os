# 登录 404 错误诊断与修复 - 20260317

## 问题描述

用户登录页面报 404 错误：
```
Cannot POST /api/auth/login
```

## 根本原因

**运行的是简化版后端（main-simple-with-ai.ts），缺少认证路由**

### 问题分析

1. ✅ **前端请求正确**：`http://localhost:3006/api/auth/login`
2. ✅ **后端有全局前缀**：`app.setGlobalPrefix('api')` (main.ts:51)
3. ✅ **AuthController 已定义**：`@Controller('auth')` + `@Post('login')`
4. ✅ **AuthModule 已导入**：AppModule 中已导入 AuthModule
5. ❌ **运行了错误的版本**：当前运行的是 `main-simple-with-ai.ts`（没有 auth 路由）

### 验证证据

```bash
# 测试登录接口
curl -X POST http://localhost:3006/api/auth/login
# 返回: 404 Not Found

# 测试健康检查
curl http://localhost:3006/health
# 返回: {"status":"ok"} - 说明后端在运行，但是简化版
```

## 已完成的修复

### 1. ✅ 修复 pdf-parse 导入问题

**问题**：pdf-parse 2.x 版本改变了导出方式，导致编译错误

**修复的文件**：
- [resume-upload.service.ts](../apps/api/src/modules/resume-upload/resume-upload.service.ts)
- [resume-parser.service.ts](../apps/api/src/modules/ai/resume-parser.service.ts)
- [main-simple.ts](../apps/api/src/main-simple.ts)

**修复方法**：
```typescript
// ❌ 旧方法（不兼容）
import * as pdf from 'pdf-parse';
const data = await pdf(buffer);

// ✅ 新方法（兼容 pdf-parse 2.x）
const pdfParse = await import('pdf-parse');
const data = await (pdfParse as any).default(buffer);
```

### 2. ✅ 修复 Prisma 类型问题

**文件**：[candidates.service.ts](../apps/api/src/modules/candidates/candidates.service.ts)

**修复**：
```typescript
data: data as any, // 类型断言修复 Prisma 类型检查
```

### 3. ✅ 启用 ResumeUploadModule

**文件**：[app.module.ts](../apps/api/src/app.module.ts)

```typescript
ResumeUploadModule,  // ✅ 已启用 - 修复了 pdf-parse 兼容性问题
```
