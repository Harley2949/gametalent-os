# 代码风格规范

## 概述

本项目使用 ESLint 和 Prettier 来确保代码风格一致性和质量。

## 配置文件

- `.eslintrc.json` - ESLint 配置
- `.prettierrc` - Prettier 配置
- `.prettierignore` - Prettier 忽略文件

## 代码风格规则

### 1. 基础规则

| 规则 | 配置 | 说明 |
|------|------|------|
| 分号 | 必须使用 | `;` |
| 引号 | 单引号 | `'string'` |
| 尾随逗号 | ES5 | `[1, 2, ]` |
| 缩进 | 2 空格 | 不使用制表符 |
| 行宽 | 100 字符 | 超出自动换行 |
| 结束符 | LF (Unix) | 不使用 CRLF |

### 2. TypeScript 规则

```typescript
// ✅ 推荐
const userName: string = 'John';
const userAge: number = 25;

// ❌ 避免
const userName = 'John'; // 缺少类型注解
const data: any = fetchData(); // 避免使用 any

// ✅ 使用下划线忽略未使用变量
const _unused = 'value';
```

### 3. React/Next.js 规则

```tsx
// ✅ 组件命名：PascalCase
export function UserProfile() {
  return <div>User Profile</div>;
}

// ✅ 自定义 Hook：use 开头
export function useUserData() {
  // ...
}

// ✅ Props 解构
function Button({ label, onClick }: ButtonProps) {
  return <button onClick={onClick}>{label}</button>;
}

// ❌ 避免
function button({ label, onClick }) { // 组件命名错误
  // ...
}
```

### 4. Import 顺序

```typescript
// 1. 内置模块
import { useState } from 'react';
import { NextPage } from 'next';

// 2. 第三方模块
import { Button } from '@/components/ui/button';

// 3. 内部模块（相对路径）
import { TableSkeleton } from '@/components/shared';
import { fetchJobs } from '@/lib/api';

// 4. 类型导入
import type { Job, Candidate } from '@/types';

// 5. CSS 样式
import styles from './page.module.css';
```

### 5. 文件命名

| 类型 | 命名规范 | 示例 |
|------|---------|------|
| 组件 | PascalCase | `UserProfile.tsx` |
| 工具函数 | camelCase | `formatDate.ts` |
| 类型定义 | camelCase | `jobTypes.ts` |
| 常量 | UPPER_SNAKE_CASE | `API_CONSTANTS.ts` |
| Hook | camelCase (use 开头) | `useUserData.ts` |
| 样式 | camelCase | `buttonStyles.css` |

## 可用命令

### 格式化代码

```bash
# 格式化所有文件
pnpm format

# 检查格式（不修改文件）
pnpm format:check
```

### 代码检查

```bash
# 运行 ESLint
pnpm lint

# 自动修复可修复的问题
pnpm lint:fix

# 类型检查
pnpm type-check
```

### 组合命令

```bash
# 格式化 + Lint 检查
pnpm format && pnpm lint

# 完整检查（CI 使用）
pnpm format:check && pnpm type-check && pnpm lint
```

## 编辑器配置

### VS Code

推荐安装以下扩展：

1. ESLint
2. Prettier - Code formatter
3. TypeScript Vue Plugin (Volar)

在 `.vscode/settings.json` 中添加：

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

### WebStorm / IntelliJ IDEA

1. 打开 Settings/Preferences
2. 搜索 "Prettier"
3. 启用 "On save" 格式化
4. 搜索 "ESLint"
5. 启用 "Automatic ESLint configuration"

## Git Hooks（可选）

可以使用 husky 和 lint-staged 在提交前自动格式化：

```bash
# 安装依赖
pnpm add -D husky lint-staged

# 初始化 husky
pnpm husky install
pnpm husky add .husky/pre-commit "pnpm lint-staged"
```

在 `package.json` 中添加：

```json
{
  "lint-staged": {
    "*.{ts,tsx,js,jsx}": [
      "eslint --fix",
      "prettier --write"
    ],
    "*.{json,css,scss,md}": [
      "prettier --write"
    ]
  }
}
```

## 常见问题

### Q: 为什么我的代码格式化后还是报错？

A: 请确保已安装所有依赖：

```bash
pnpm install
```

### Q: 如何禁用某行代码的检查？

A: 使用注释：

```typescript
// eslint-disable-next-line no-console
console.log('This line will not be checked');
```

### Q: 如何临时禁用整个文件的检查？

A: 在文件顶部添加：

```typescript
/* eslint-disable */
```

## 代码审查清单

在提交代码前，请确保：

- [ ] 运行 `pnpm format` 格式化代码
- [ ] 运行 `pnpm lint` 检查代码质量
- [ ] 运行 `pnpm type-check` 检查类型错误
- [ ] 所有测试通过 `pnpm test`
- [ ] 代码符合团队的命名规范
- [ ] 添加了必要的注释和文档
