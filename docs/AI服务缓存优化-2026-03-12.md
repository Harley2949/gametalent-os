## 🎯 缓存策略

### TTL 配置（生存时间）

| 缓存类型 | TTL | 原因 |
|---------|-----|------|
| 匹配分数 | 1 小时 | 职位/候选人更新频率中等 |
| 技能提取 | 2 小时 | 简历内容相对稳定 |
| 竞品分析 | 24 小时 | 竞品信息变化很少 |

### 缓存键格式

```
match:{candidateId}:{jobId}      # 匹配分数
skills:{resumeId}                 # 技能提取
competitor:{resumeId}             # 竞品分析
```

## 🚀 使用示例

### 1. 基础使用（自动缓存）

```typescript
// 第一次调用：执行 Ollama API（3-10秒）
const result1 = await aiService.calculateMatchScore('candidate-1', 'job-1');

// 第二次调用：从缓存获取（<100ms）
const result2 = await aiService.calculateMatchScore('candidate-1', 'job-1');
```

### 2. 缓存失效

当数据更新时，手动清除缓存：

```typescript
// 当候选人信息更新时
await aiService.invalidateCandidateCache(candidateId);

// 当职位信息更新时
await aiService.invalidateJobCache(jobId);

// 当简历更新时
await aiService.invalidateResumeCache(resumeId);
```

### 3. 直接使用缓存服务

```typescript
import { CacheService } from './modules/cache/cache.service';

@Injectable()
export class MyService {
  constructor(private cacheService: CacheService) {}

  async expensiveOperation() {
    // 使用 getOrSet 简化缓存逻辑
    return this.cacheService.getOrSet(
      'my-expensive-key',
      () => this.doHeavyWork(),
      3600 // 1小时
    );
  }
}
```

## 🛠️ 高级配置

### 使用 Redis（生产环境推荐）

#### 1. 安装 Redis 适配器

```bash
pnpm add cache-manager-redis-store
```

#### 2. 更新缓存模块

```typescript
// apps/api/src/modules/cache/cache.module.ts
import * as redisStore from 'cache-manager-redis-store';

@Global()
@Module({
  imports: [
    CacheModule.register({
      store: redisStore,
      host: process.env.REDIS_HOST || 'localhost',
      port: parseInt(process.env.REDIS_PORT || '6379'),
      ttl: 3600,
      max: 10000, // Redis 可以存储更多
    }),
  ],
  // ...
})
export class CacheModule {}
```

#### 3. 更新 .env

```env
# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_password # 可选
```

## 🔍 故障排除

### 问题 1：缓存不生效

**症状**：每次调用都需要 3-10 秒

**解决方案**：
1. 检查 CacheModule 是否在 app.module.ts 中导入
2. 确认缓存键格式正确
3. 查看日志确认是否触发缓存命中

### 问题 2：类型错误

**症状**：`CacheService` 无法注入

**解决方案**：
```bash
# 重新构建项目
cd apps/api
pnpm build
```

### 问题 3：内存占用过高

**症状**：内存占用持续增长

**解决方案**：
1. 减少 `max` 配置（默认 1000）
2. 缩短 TTL 时间
3. 使用 Redis 替代内存缓存

## 🎯 下一步优化建议

### 1. 添加缓存预热（Cache Warm-up）

在应用启动时预加载常用数据：

```typescript
async onModuleInit() {
  // 预加载热门职位的匹配分数
  const hotJobs = await this.prisma.job.findMany({
    where: { status: 'ACTIVE' },
    orderBy: { viewCount: 'desc' },
    take: 10,
  });

  for (const job of hotJobs) {
    // 后台预热，不阻塞启动
    this.preloadJobCache(job.id).catch(err => {
      this.logger.error(`预热失败 [${job.id}]:`, err);
    });
  }
}
```
