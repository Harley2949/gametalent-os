## 二、权限体系设计（简化版 - 3角色）

### 2.1 角色定义

我们采用**3角色极简方案**，清晰明了，易于管理：

| 角色 | 名称 | 说明 | 典型人员 |
|------|------|------|----------|
| **ADMIN** | 系统管理员 | 拥有所有权限，负责系统维护和用户管理 | 系统管理员 |
| **RECRUITER** | HR招聘人员 | 负责日常招聘操作 | 招聘专员、HRBP |
| **INTERVIEWER** | 面试官 | 只参与面试评估，查看有限信息 | 技术面试官、美术面试官 |

> **极简说明**：
> 1. 只有3个角色，分工清晰
> 2. HR（RECRUITER）负责所有招聘操作
> 3. 面试官（INTERVIEWER）只负责评估
> 4. 业务部门通过"职位负责人"参与招聘，不需要独立角色

- 管理内推流程

#### INTERVIEWER - 面试官
**职责**：参与面试，提供评估意见
**包括**：技术专家、美术专家、资深员工

### 2.4 内推渠道功能设计

内推（内部推荐）是高效的招聘方式，系统需要支持完整的内推流程。

#### 内推流程
```
1. 员工生成内推码/链接
   ↓
2. 员工分享给候选人
   ↓
3. 候选人通过内推链接投递简历
   ↓
4. 系统自动关联推荐人
   ↓
5. 候选人入职后，推荐人获得奖励
```

#### 数据库设计
```prisma
// 内推推荐记录
model Referral {
  id              String   @id @default(cuid())
  code            String   @unique  // 内推码
  referrerId      String            // 推荐人ID（公司员工）

  // 内推配置
  position        String?          // 推荐职位（可选，为空则可推荐任何职位）
  maxUses         Int      @default(1)  // 最大使用次数
  usedCount       Int      @default(0)  // 已使用次数
  expiresAt       DateTime?         // 过期时间

  // 奖励设置
  rewardAmount    Float?            // 推荐成功奖励金额
  rewardType      String?           // 奖励类型：现金/礼品/假期等

  // 统计
  viewCount       Int      @default(0)  // 浏览次数
  clickCount      Int      @default(0)  // 点击次数

  status          String   @default("ACTIVE")  // ACTIVE/EXPIRED/USED

  referrer        User     @relation("Referrer", fields: [referrerId], references: [id])
  candidates      Candidate[]  // 通过此内推码投递的候选人

  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  @@map("referrals")
}

// 候选人与内推关联
model Candidate {
  // ... 其他字段

  referralId      String?   // 内推记录ID
  referral        Referral? @relation(fields: [referralId], references: [id])

  // 内推奖励状态
  rewardStatus    String?   // PENDING/PAILED/PAID

  @@map("candidates")
}
```

#### 功能特性

**1. 内推码生成**
- HR 批量生成内推码
- 为特定部门/职位生成内推码
- 设置有效期和使用次数
- 设置推荐奖励

**2. 内推链接分享**
- 生成专属内推链接
- 支持二维码分享
- 链接可包含职位信息
- 追踪来源渠道

**3. 候选人投递**
- 候选人通过内推链接访问
- 自动关联推荐人
- 显示"你是被XXX推荐"
- 自动标记来源为"内推"

**4. 推荐管理**
- 查看所有内推记录
- 追踪推荐进度
- 查看推荐人榜单
- 发放推荐奖励

**5. 数据统计**
- 各部门内推人数
- 内推入职率
- 内推成本分析
- 推荐人排行

### 4.1 新增权限守卫

创建 `apps/api/src/modules/auth/guards/roles.guard.ts`：

```typescript
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { UserRole } from '@gametalent/db';

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const requiredRoles = this.reflector.getAllAndOverride<UserRole[]>('roles', [
      context.getHandler(),
      context.getClass(),
    ]);

    if (!requiredRoles) {
      return true;
    }

    const { user } = context.switchToHttp().getRequest();
    return requiredRoles.some((role) => user.role === role);
  }
}
```

创建 `apps/api/src/modules/auth/guards/permissions.guard.ts`：

```typescript
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { PrismaService } from '../prisma/prisma.service';

@Injectable()
export class PermissionsGuard implements CanActivate {
  constructor(
    private reflector: Reflector,
    private prisma: PrismaService,
  ) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const requiredPermissions = this.reflector.getAllAndOverride<string[]>('permissions', [
      context.getHandler(),
      context.getClass(),
    ]);

    if (!requiredPermissions) {
      return true;
    }

    const { user } = context.switchToHttp().getRequest();

    // 获取用户的所有权限
    const roleMappings = await this.prisma.userRoleMapping.findMany({
      where: { userId: user.userId },
      include: {
        role: {
          include: {
            permissions: {
              include: {
                permission: true,
              },
            },
          },
        },
      },
    });

    const userPermissions = roleMappings.flatMap(rm =>
      rm.role.permissions.map(rp => rp.permission.name)
    );

    return requiredPermissions.some(permission => userPermissions.includes(permission));
  }
}
```

### 5.4 路由守卫

创建 `apps/web/src/middleware.ts`：

```typescript
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(request: NextRequest) {
  const token = request.cookies.get('access_token');
  const path = request.nextUrl.pathname;

  // 公开页面
  const publicPaths = ['/login', '/register'];
  if (publicPaths.includes(path)) {
    return NextResponse.next();
  }

  // 需要登录的页面
  if (!token) {
    return NextResponse.redirect(new URL('/login', request.url));
  }

  return NextResponse.next();
}

export const config = {
  matcher: ['/candidates/:path*', '/jobs/:path*', '/admin/:path*'],
};
```

   - HttpOnly Cookie 存储 Token

3. **审计日志**：
   - 记录用户登录/登出
   - 记录敏感操作（删除、权限变更）
   - 提供审计日志查询功能

   - 登录失败次数限制（防暴力破解）
   - IP 白名单（可选）
   - 双因素认证（2FA，可选）
