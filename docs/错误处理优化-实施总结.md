### 1. **统一错误类系统** ✅

创建了完整的错误类型层次结构：

**文件**：[apps/api/src/common/errors/app.error.ts](apps/api/src/common/errors/app.error.ts)

**功能**：
- ✅ AppError 基类（包含所有必要属性）
- ✅ 错误代码枚举（20+ 种错误类型）
- ✅ 静态工厂方法（快速创建常见错误）
- ✅ 业务逻辑错误类（BusinessRuleError、StateTransitionError）
- ✅ 错误详情字段（传递额外上下文）
- ✅ 操作性标记（区分预期错误和意外错误）

**使用示例**：
```typescript
// 404 错误
throw AppError.notFound('候选人', { id: '123' });

// 400 验证错误
throw AppError.validation('邮箱格式无效', {
  field: 'email',
  constraints: { isEmail: '邮箱格式不正确' }
});

// 业务规则错误
throw new BusinessRuleError('不能删除有关联数据的候选人');
```

## 🔍 错误处理对比

### 之前（❌ 混乱）

```typescript
// Service 中
if (!candidate) {
  throw new Error('Candidate not found');
}

// Controller 中
@Get(':id')
async findOne(@Param('id') id: string) {
  try {
    return this.service.findOne(id);
  } catch (error) {
    throw new BadRequestException(error.message);
  }
}

// 响应格式不统一
// 有时是 { message: "..." }
// 有时是 { error: "..." }
// 有时是 HTML 错误页面
```

### 现在（✅ 统一）

```typescript
// Service 中
if (!candidate) {
  throw AppError.notFound('候选人', { id });
}

// 自动转换为
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "候选人不存在",
    "details": { "id": "123" },
    "timestamp": "2026-03-17T10:30:00.000Z",
    "path": "/api/candidates/123",

  }
```

## 🎓 最佳实践总结

### 错误类型选择

- **AppError.notFound()**：资源不存在（候选人、职位等）
- **AppError.validation()**：用户输入验证失败
- **AppError.unauthorized()**：用户未登录
- **AppError.forbidden()**：用户无权限
- **BusinessRuleError**：违反业务规则
- **AppError.internal()**：服务器内部错误

### 错误消息编写

✅ 好的消息：
- "邮箱格式不正确，请输入有效的邮箱地址"
- "该邮箱已被注册，请使用其他邮箱或登录"
- "登录已过期，请重新登录"

❌ 不好的消息：
- "错误"
- "Invalid input"
- "Database query failed"

### 日志级别

- **error**：5xx 错误、未处理的异常
- **warn**：4xx 错误、业务逻辑错误
- **info**：正常操作、重要事件
- **debug**：调试信息
