# 智能面试排期功能 - 实现文档

## 概述

本文档描述了 GameTalent OS 智能面试排期功能的完整实现。该功能已在第一阶段技术架构中完成设计和编码。

## 一、数据库设计

### 1.1 新增数据表

#### UserAvailability - 用户可用时间配置
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| userId | String | 用户ID (唯一) |
| workDays | Int[] | 工作日 [1-5] 表示周一到周五 |
| workStartAt | String | 工作开始时间 "09:00" |
| workEndAt | String | 工作结束时间 "18:00" |
| breakStartAt | String? | 午休开始 "12:00" |
| breakEndAt | String? | 午休结束 "13:00" |
| timezone | String | 时区 "Asia/Shanghai" |
| minNoticeHours | Int | 最小预约提前时间 |
| maxBookingDays | Int | 最大预约天数 |
| blackoutDates | DateTime[] | 不可用日期（假期、出差） |

#### CalendarIntegration - 外部日历集成
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| userId | String | 用户ID |
| provider | String | 提供商 (GOOGLE/OUTLOOK/FEISHU) |
| accessToken | String | 访问令牌 |
| refreshToken | String? | 刷新令牌 |
| calendarId | String | 要同步的日历ID |
| autoSync | Boolean | 是否自动同步 |
| lastSyncAt | DateTime? | 最后同步时间 |

#### InterviewInvitation - 面试邀请跟踪
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| interviewId | String | 面试ID (唯一) |
| status | String | 状态 (PENDING/ACCEPTED/DECLINED) |
| respondedAt | DateTime? | 响应时间 |
| sentAt | DateTime | 发送时间 |
| expiresAt | DateTime | 过期时间 |
| remindersSent | Int | 已发提醒数 |

#### InterviewReminder - 面试提醒队列
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| interviewId | String | 面试ID |
| type | String | 类型 (BEFORE_24H/BEFORE_1H/FOLLOW_UP) |
| scheduledAt | DateTime | 计划发送时间 |
| status | String | 状态 (PENDING/SENT/FAILED) |

#### SuggestedTimeSlot - 推荐时段缓存
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 主键 |
| interviewId | String? | 面试ID |
| suggestedAt | DateTime | 推荐时间 |
| duration | Int | 时长(分钟) |
| interviewerIds | String[] | 面试官ID列表 |
| score | Float? | 推荐评分 |
| reasons | String[] | 推荐理由 |

## 二、API 接口

### 2.1 获取智能排期建议
```http
POST /scheduling/suggestions
Content-Type: application/json

{
  "applicationId": "string",
  "interviewerIds": ["string"],
  "duration": 60,
  "startDate": "2024-01-15T00:00:00Z",
  "endDate": "2024-01-29T00:00:00Z",
  "candidateTimeZone": "Asia/Shanghai",
  "preferredTimes": ["morning", "afternoon"]
}
```

**响应示例：**
```json
[
  {
    "start": "2024-01-16T10:00:00Z",
    "end": "2024-01-16T11:00:00Z",
    "score": 115,
    "reasons": ["morning_time", "soon_availability"]
  },
  {
    "start": "2024-01-16T14:00:00Z",
    "end": "2024-01-16T15:00:00Z",
    "score": 95,
    "reasons": []
  }
]
```

### 2.2 获取用户可用时间配置
```http
GET /scheduling/availability/:userId
```

### 2.3 更新用户可用时间配置
```http
PUT /scheduling/availability/:userId
Content-Type: application/json

{
  "workDays": [1, 2, 3, 4, 5],
  "workStartAt": "09:00",
  "workEndAt": "18:00",
  "breakStartAt": "12:00",
  "breakEndAt": "13:00",
  "timezone": "Asia/Shanghai",
  "minNoticeHours": 2,
  "maxBookingDays": 30,
  "blackoutDates": ["2024-02-10T00:00:00Z"]
}
```

### 2.4 获取用户空闲时段
```http
GET /scheduling/slots/:userId?startDate=2024-01-15&endDate=2024-01-22
```

### 2.5 智能排期 - 创建面试
```http
POST /scheduling/schedule
Content-Type: application/json

{
  "applicationId": "string",
  "interviewerId": "string",
  "title": "技术面试 - Unity开发工程师",
  "description": "第一轮技术面试",
  "type": "TECHNICAL",
  "stage": "FIRST_ROUND",
  "scheduledAt": "2024-01-16T10:00:00Z",
  "duration": 60,
  "location": "https://meet.google.com/xxx"
}
```

### 2.6 改期
```http
POST /scheduling/reschedule/:interviewId
Content-Type: application/json

{
  "scheduledAt": "2024-01-17T14:00:00Z",
  "reason": "候选人临时有事"
}
```

### 2.7 候选人确认面试
```http
POST /scheduling/confirm/:interviewId
```

## 三、核心算法

### 3.1 时间推荐算法流程

```
1. 获取所有面试官的可用时间段
   └──> getInterviewersAvailability()

2. 计算所有面试官的时间交集
   └──> findOverlappingSlots()
       └──> intersectSlots() - 两两求交集

3. 根据偏好时间过滤
   └──> filterByPreferredTimes()

4. 对每个时段进行评分排序
   └──> scoreTimeSlots()
       ├── 上午时段 +10 分
       ├── 工作日内 +5 分
       ├── 3天内 +10 分
       ├── 超过7天 -5 分
       └── 早晚时段 -10 分

5. 返回 Top 3 推荐结果
```

### 3.2 时间冲突检测

系统会自动检测以下冲突：
- 与已有面试时间重叠
- 非工作时间
- 午休时间
- 用户设置的不可用日期

## 四、自动化功能

### 4.1 自动提醒调度

面试创建后，系统自动调度以下提醒：

| 提醒类型 | 触发时间 | 状态 |
|---------|---------|------|
| BEFORE_24H | 面试前24小时 | 自动发送 |
| BEFORE_1H | 面试前1小时 | 自动发送 |
| FOLLOW_UP | 面试结束后1小时 | 发送反馈收集提醒 |

### 4.2 邀请有效期管理

- 面试邀请默认 **48小时** 有效期
- 过期未响应可自动取消或重发
- 支持候选人在线确认/拒绝

### 4.3 改期处理流程

```
1. HR 选择新时间
   ↓
2. 系统更新面试时间
   ↓
3. 取消所有待发送的提醒
   ↓
4. 重置邀请状态为 PENDING
   ↓
5. 重新调度提醒任务
   ↓
6. 发送改期通知给候选人和面试官
```

## 十、FAQ

**Q: 如何设置不同部门的不同工作时间？**
A: 可以为每个用户单独配置 UserAvailability，或创建部门模板批量应用。

**Q: 支持跨时区排期吗？**
A: 是的，系统支持时区转换，可以自动处理候选人不同时区的情况。

**Q: 如何处理面试官临时有事？**
A: 可以在 UserAvailability 中添加 blackoutDates，或直接使用改期功能。

**Q: 候选人可以自己选择时间吗？**
A: 可以，通过候选人门户展示可用时段让候选人选择，然后调用 confirm 接口。
