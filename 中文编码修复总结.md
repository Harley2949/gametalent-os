# 中文乱码修复总结

## 已修复的问题

### 1. 数据库连接编码 ✅
**文件**: `.env`
**修改**: 添加了 `client_encoding=UTF8` 参数
```bash
# 修改前
DATABASE_URL="postgresql://...?schema=public"

# 修改后
DATABASE_URL="postgresql://...?schema=public&client_encoding=UTF8"
```

### 2. API 响应头编码 ✅
**文件**: `apps/api/src/main.ts`
**修改**: 添加了 UTF-8 响应头中间件
```typescript
// 设置 UTF-8 编码支持，防止中文乱码
app.use((req, res, next) => {
  res.setHeader('Content-Type', 'application/json; charset=utf-8');
  next();
});
```

### 3. 异常过滤器编码 ✅
**文件**: `apps/api/src/common/filters/http-exception.filter.ts`
**修改**: 确保错误响应也使用 UTF-8 编码

## 可能的乱码场景和解决方案

### 场景1: 控制台输出乱码
**原因**: Windows 控制台默认编码可能不是 UTF-8
**解决**:
```bash
# PowerShell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8

# CMD
chcp 65001
```

### 场景2: 浏览器显示乱码
**检查步骤**:
1. 按 F12 打开开发者工具
2. 查看网络请求的响应头，确认 `Content-Type: application/json; charset=utf-8`
3. 查看数据库中存储的数据是否正确

### 场景3: 特定字段乱码
如果是某个特定字段乱码，可能是：
1. 该字段在数据库中的类型不支持中文（应使用 `Text` 或 `VarChar`）
2. 数据迁移时编码未正确转换
