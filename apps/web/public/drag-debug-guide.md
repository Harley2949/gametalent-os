## 🧪 测试步骤

### **步骤1：刷新页面**
1. 在浏览器中访问：`http://localhost:3000/applications?view=kanban`
2. 按 `Ctrl+Shift+R`（硬刷新）确保加载最新代码
3. 等待页面完全加载

### **步骤2：查找调试面板**
- 页面右下角应该出现一个**黑色标题栏的面板**
- 标题显示："🔍 拖拽调试面板"

### **步骤3：观察调试面板**
调试面板显示：
- **拖拽状态**：空闲 / 拖拽中
- **拖拽卡片**：显示当前拖拽的卡片ID
- **目标列**：显示当前悬停的列
- **鼠标位置**：实时X/Y坐标
- **事件日志**：所有拖拽事件的实时记录

### **步骤4：测试拖拽功能**

#### **测试4.1：开始拖拽**
1. 点击并按住任意应聘卡片（如"李思琪"）
2. 调试面板应显示：
   - 拖拽状态变为"拖拽中"（绿色）
   - 显示拖拽卡片ID
   - 日志显示："🎯 拖拽开始"

#### **测试4.2：拖动到目标列**
1. 保持按住鼠标，将卡片拖到其他列（如"筛选中"）
2. 调试面板应显示：
   - 目标列更新（橙色）
   - 日志显示："📍 拖拽经过"
   - 目标列应该**高亮**（蓝色边框+放大）

#### **测试4.3：放置卡片**
1. 在目标列上松开鼠标
2. 调试面板应显示：
   - 拖拽状态恢复为"空闲"
   - 日志显示："🎯 拖拽放置"
   - 页面应显示成功提示："✅ 状态更新成功"
3. 卡片应该移动到新列

### **步骤5：使用调试按钮**

调试面板底部有两个测试按钮：

#### **按钮1：检查拖拽元素**
- 点击后会弹出提示
- 显示页面上找到的可拖拽元素数量
- 应该显示：找到 X 个可拖拽元素

#### **按钮2：检查登录状态**
- 点击后弹出提示
- 显示是否已登录

## 🐛 可能的问题和解决方案

### **问题A：看不到调试面板**

**原因**：页面缓存或代码未重新编译

**解决方案**：
1. 按 `Ctrl+Shift+R` 硬刷新
2. 或清除浏览器缓存后刷新
3. 检查页面是否完全加载

### **问题B：调试面板显示但拖拽不工作**

**检查1：点击"检查拖拽元素"按钮**
- 应该显示找到的可拖拽元素数量
- 如果显示 0，说明卡片没有正确设置 `draggable` 属性

**检查2：尝试拖拽卡片**
- 观察调试面板的"拖拽状态"
- 如果一直是"空闲"，说明拖拽事件没有触发
- 可能原因：CSS阻止、事件被拦截、浏览器不支持

**检查3：点击"检查登录状态"按钮**
- 如果显示"未登录"，拖拽功能会失败
- 需要先登录才能使用拖拽功能

### **问题C：拖拽开始但不更新状态**

**可能原因**：
1. 未登录或token过期
2. API调用失败
3. 状态转换规则限制

**调试方法**：
1. 确保已登录
2. 观察调试面板的日志
3. 检查是否有错误提示

## 📊 调试面板日志说明

日志类型：
- 🎯 绿色 = 拖拽开始/放置
- 📍 橙色 = 拖拽经过
- ⬅️ 黄色 = 拖拽离开
- ✅ 蓝色 = 拖拽结束
- ❌ 红色 = 错误

## 🎯 预期结果

**正常情况下，拖拽过程应该是：**

1. 点击卡片 → 状态变为"拖拽中" → 日志："🎯 拖拽开始"
2. 拖动到目标列 → 显示"目标列" → 日志："📍 拖拽经过" → 目标列高亮
3. 松开鼠标 → 状态变为"空闲" → 日志："🎯 拖拽放置" → 显示成功提示
4. 卡片移动到新列 → 状态更新成功
